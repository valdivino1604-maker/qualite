# Site Qualitê Consultoria e Projetos Site institucional estático, responsivo e pronto para publicar no GitHub e Cloudflare Pages. ## Arquivos principais - `index.html` - página inicial - `sobre.html` - história, missão, visão e valores - `servicos.html` - serviços oferecidos - `projetos.html` - aplicações da marca e projetos - `area-cliente.html` - área do cliente com login simulado - `contato.html` - formulário com envio por WhatsApp - `assets/css/style.css` - identidade visual - `assets/js/script.js` - menu, filtros, WhatsApp e área do cliente - `assets/data/clientes.json` - base simulada de clientes - `assets/docs/` - documentos de demonstração ## Login de demonstração da área do cliente - E-mail: `cliente@qualite.com.br` - Senha: `123456` - Código: `QUALITE001` ## Configurar WhatsApp Abra o arquivo `assets/js/script.js` e altere: ```js whatsappNumber: '5562999999999' ``` Substitua pelo número oficial com DDI e DDD. ## Como subir no GitHub 1. Crie um repositório no GitHub. 2. Envie todos os arquivos deste pacote para o repositório. 3. Faça commit na branch `main`. 4. Verifique se o arquivo `index.html` está na raiz do projeto. Comandos opcionais: ```bash git init git add . git commit -m "Site Qualite Consultoria e Projetos" git branch -M main git remote add origin https://github.com/SEU-USUARIO/NOME-DO-REPOSITORIO.git git push -u origin main ``` ## Como publicar no Cloudflare Pages 1. Acesse o Cloudflare. 2. Vá em **Workers & Pages**. 3. Clique em **Create application** ou **Create project**. 4. Escolha **Pages**. 5. Conecte ao GitHub. 6. Selecione o repositório do site. 7. Configure: - Framework preset: `None` - Build command: deixar vazio - Output directory: `/` 8. Clique em **Deploy**. ## Melhorias futuras - Login real com Firebase, Supabase ou backend próprio. - Upload de documentos por administrador. - Painel administrativo. - Integração com e-mail. - Integração com pagamentos. - Controle real de status dos serviços. - Geração automática de propostas.
